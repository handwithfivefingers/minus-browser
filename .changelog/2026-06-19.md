# Changelog — 2026-06-19

## History: stale title & favicon in entries

`did-navigate` / `did-navigate-in-page` fire before `page-title-updated` / `page-favicon-updated`, so history entries were permanently stored with the **previous page's title** and an empty favicon.

**Fix:** Added `History.updateEntryMetadata(url, title?, favicon?)` to `src/core/controller/history/historyController.ts:94`. Called from `updateTitle()` and `updateFavicon()` in `src/features/tabs/models/tab.ts` to retroactively correct the entry when metadata becomes available.

## History: `updateUrl` sent URL as `title` to renderer

`src/features/tabs/models/tab.ts:284` — `updateUrl` called `persistInformationToRenderer({ title: this.url })`, which wrote the URL into the renderer's `title` field, corrupting the tab state.

**Fix:** Changed to `{ url: this.url }`.

## History: `did-navigate` handler broken on Electron 42

`did-navigate` event removed the `isMainFrame` parameter in Electron 42. The handler at `src/features/tabs/models/tab.ts:221` declared `(_event, url, _httpStatus, _httpStatusText, isMainFrame)` but the event passes only 4 params, so `isMainFrame` was always `undefined` and `addEntry` never executed.

**Fix:** Removed the `isMainFrame` parameter and guard. `did-navigate` already fires only for main frame navigations per current Electron docs.

## History page: no real-time updates

The History UI fetched entries once on mount with no refresh mechanism.

**Fix:** Added a 3-second polling interval and a manual refresh button (`IconRefresh`) in `src/features/ui/pages/history/index.tsx`.

## ViewController: double `createView` for hibernated tabs

`handleShowViewById` precomputed `wasNotAlive` before `show()`. For hibernated tabs, `show()` → `wake()` already calls `createView()`, then the stale `wasNotAlive` flag triggered a second `createView()`, leaking the first WebContentsView and causing a redundant navigation.

**Fix:** Check `currentTab.isAlive` **after** `show()` in `src/core/controller/viewController.ts:194-198`. Only call `createView()` if the tab is still not alive after waking.

## [Unreleased]

### Added

- **Hibernate Settings Panel**: New settings tab under Settings → Hibernate with four modes:
  - Fast: Hibernate after 15 minutes
  - Normal: Hibernate after 1 hour (default)
  - Slow: Hibernate after 4 hours
  - Custom: User-defined timeout (1-1440 minutes)
- **Per-tab Hibernation Protection**: Snowflake toggle in each tab's header bar. When active (cyan), the tab is excluded from auto-hibernation regardless of inactivity duration.
- **IPC**: `TOGGLE_PREVENT_HIBERNATE` invoke channel for toggling per-tab protection.

### Changed

- **Header URL bar**: Now displays page `title` instead of `url` by default, showing `url` only as fallback.
- **Spotlight search trigger**: Passes `title` instead of `url` to the spotlight overlay query.
- **TabController**: Hibernate timer now checks `preventHibernate` flag per tab. Hibernate delay is configurable via `setHibernateMode()` instead of hardcoded 1 hour.
- **CustomApp**: Reads tab state from `tabs` array by ID for reactive updates (no more stale `activeTab` reference).

### Technical

- `ITab` interface extended with `preventHibernate?: boolean`
- `IUserInterface` extended with `hibernateMode` and `hibernateCustomMinutes`
- `TabController.interfaceStore` added for hibernate settings persistence
- Settings automatically propagate to `TabController` on save and app start
